## OVERVIEW
PicoClaw is an ultra-lightweight AI Agent written in Go, targeting $10 hardware with <10MB RAM. Multi-component: CLI tool (`picoclaw`), Web Launcher (Fyne + React), and 18+ chat channel integrations.

## CONVENTIONS (THIS PROJECT)
- **Go formatting**: `gofmt`, `goimports`, `gofumpt`, `golines` (120 char width)
- **Frontend**: 2-space indent, ESLint + Prettier via `pnpm check`
- **Test files**: Co-located `*_test.go`, table-driven pattern with `t.Run`
- **Integration tests**: `*_integration_test.go` suffix
- **Build tags**: `goolm` (default), `stdjson`, `whatsapp_native`, `bedrock`
- **Accept interfaces, return structs** - core design principle

## ANTI-PATTERNS (THIS PROJECT)
- **NEVER** use `interface{}` - use `any`
- **NEVER** commit without running `make check` (lint + vet + test)
- **NEVER** hardcode sensitive data - use `.security.yml` pattern
- **DO NOT** modify `workspace/` files directly during development

## UNIQUE STYLES
- **AI-bootstrapped**: 95% code generated by AI, human-reviewed
- **Workspace config**: `AGENT.md` + `SOUL.md` + `SKILL.md` define agent behavior (Markdown-based)
- **Seahorse memory**: DAG-structured SQLite with FTS5 CJK support
- **Multi-arch builds**: MIPS, RISC-V, LoongArch, ARM, x86 in `Makefile`
- **MIPS ELF patching**: `patchelf`-incompatible e_flags modification via `dd`

## COMMANDS
```bash
make build           # Build picoclaw for current platform
make build-launcher  # Build Web UI Launcher
make test            # Run Go tests (excl. web) + web tests
make lint            # golangci-lint run
make fix             # Auto-fix linting issues
make deps            # Download + verify Go modules
make docker-build    # Alpine-based Docker image
```

## NOTES
- Frontend has **no test framework** (no jest/vitest configured)
- Web frontend: Vite + React 19 + TypeScript + shadcn/ui
- Binary <20MB, RAM <10MB (goal); current builds may use 10-20MB
- Security: sensitive data in `.security.yml`, config.json stores only insensitive values
